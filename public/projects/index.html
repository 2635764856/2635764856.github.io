项目作品集

项目作品集#

在我的开发生涯中，我参与了多个具有挑战性的项目。以下是我最具代表性的三个项目，它们充分展示了我在后端开发、系统设计和问题解决方面的能力。

1. 农产品智能仓储系统#

时间: 2024.07 – 2025.01 | 角色: 后端开发 | 成就: 软件著作权（登记号：2025SR0210727）

项目概述#

这是一个企业级的微服务系统，旨在为农产品仓储提供智能化管理。系统涵盖用户管理、库存管理、环境监控和预警等多个核心模块，能够处理百万级数据，支持实时监控和异步处理。

技术栈#

技术 用途
Spring Boot 微服务框架
Spring Security + JWT 用户认证与授权
MySQL 关系型数据库
Redis 缓存层
RabbitMQ 异步消息处理
WebSocket 实时数据推送
Docker & Docker Compose 容器化部署
CI/CD 自动化流程

核心功能#

库存管理模块

我设计并实现了一个支持百万级数据的库存管理系统。系统支持批次管理、保质期追踪和库存状态监控。最关键的是，我实现了 FIFO（先进先出）出库策略，确保农产品的新鲜度和质量。

权限控制系统

使用 Spring Security 和 JWT 实现了基于角色的访问控制（RBAC）。系统支持多个角色（如管理员、仓库管理员、操作员等），每个角色都有明确定义的权限。

环境监控与预警

设计了环境监控 API，能够接收来自嵌入式设备（如温度传感器、湿度传感器）的实时数据。系统通过 WebSocket 实时推送监控数据给前端，并在环境参数超出预设范围时触发预警。

性能优化

  • 集成 Redis 缓存层，减少数据库查询次数，提升系统响应速度。
  • 使用 RabbitMQ 进行异步任务处理，如数据备份、报告生成等，避免阻塞主业务流程。

容器化部署

编写了 Dockerfile 和 Docker Compose 配置文件，支持一键部署。整个系统可以在 Linux 环境中快速启动，并集成了 CI/CD 流程，实现自动化测试和部署。

个人贡献#

  • 参与微服务架构设计，确定了各个模块的职责和接口。
  • 独立开发了库存、环境监控和预警模块。
  • 设计了数据库架构，支持高并发和复杂查询。
  • 实现了权限控制和实时通信功能。
  • 负责系统的容器化和部署。

2. 基于 FS-MP1A 的元宇宙智能农业系统#

时间: 2024.06 - 2024.07 | 角色: 后端开发

项目概述#

这是一个创新性的项目，结合了物联网、元宇宙和农业的概念。系统利用 FS-MP1A 设备进行数据采集，并通过软件系统进行处理和可视化。

主要职责#

  • 设备连接：负责智能农业系统的设备配置与软件之间的连接，实现了设备数据的稳定传输。
  • 全程参与：从项目的初期设计、中期开发到最终测试，全程参与并协调不同模块之间的工作。
  • 模块协调：确保各个模块（如数据采集、处理、存储和展示）之间的无缝协作。

技术亮点#

  • 实现了稳定的设备通信协议。
  • 设计了高效的数据处理流程。
  • 确保了系统的可扩展性和可维护性。

3. 基于 Android 的智能课堂考勤系统#

时间: 2025.01 - 2025.03 | 角色: 后端开发 | 定位: 毕业项目

项目概述#

这是我的毕业项目，一个综合性的课堂管理系统。系统支持学生、教师和管理员三个端口，具备位置定位和人脸识别功能，旨在提高课堂管理的效率和准确性。

技术栈#

技术 用途
Kotlin Android 移动端开发
Node.js + Express 后端框架
MySQL 数据库
RESTful API 前后端通信
位置定位 API 地理位置服务
人脸识别 API 身份验证

核心功能#

三端应用

  • 学生端：学生可以通过 Android 应用进行签到，系统会记录其位置和时间。
  • 教师端：教师可以查看学生的签到情况，并管理课堂。
  • 管理员端：管理员可以查看全校的考勤数据，生成报告。

位置定位

系统集成了位置定位功能，确保学生在指定地点进行签到。这防止了代签现象，提高了考勤的准确性。

人脸识别

系统集成了人脸识别功能，提供了额外的身份验证层。这增强了系统的安全性。

高并发支持

系统经过校园 11 个班级（300 人）的实际教学环境测试。在高峰时段（如课程开始时），系统需要处理数百个并发请求。通过以下优化措施，系统表现出良好的并发性能和抗压能力：

  • 使用连接池优化数据库连接。
  • 实现了请求队列和限流机制。
  • 优化了数据库查询和索引。

个人贡献#

  • 后端架构：使用 Node.js 和 Express 搭建了整个后端系统。
  • API 设计：设计了符合 RESTful 标准的 API，支持学生签到、查询考勤记录等功能。
  • 数据库设计：进行了 MySQL 数据库建模，支持高并发。
  • 性能优化：通过连接池和查询优化，支持 300 人的并发访问。
  • 实际测试：参与了系统的实际测试，收集反馈并进行改进。
